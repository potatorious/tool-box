# Toolbox Lab V2

Second utility-site experiment.

The goal is to choose the tool concept and identity first, then add visual theme, CSS, content structure, technical best practices, and AdSense readiness layers.

## Build Order

1. Decide the utility concept and target user.
2. Define site identity: name, promise, tone, trust posture, and content angle.
3. Write the information architecture and page inventory.
4. Define the tool workflow and calculation or transformation rules.
5. Draft original supporting content: guide, examples, FAQ, limitations, and related resources.
6. Add policy and trust pages: About, Contact, Privacy, Terms, Editorial Policy, and Disclaimer where needed.
7. Apply theme and CSS after the content model is stable.
8. Verify SEO metadata, accessibility, mobile layout, robots.txt, sitemap.xml, and deployment checklist.

## Candidate Tool Directions

- Salary and hourly-rate comparison tool
- Freelance quote sanity checker
- Subscription cost annualizer
- Unit price comparison calculator
- Travel budget splitter
- Study time planner
- Date interval and deadline calculator
- Age calculator for Korean, international, and year-based age

## Selection Criteria

- The tool solves a real repeated problem.
- The result needs explanation, not just a one-line answer.
- The topic can support original guide content, examples, and FAQs.
- The page avoids high-risk claims in medical, legal, tax, or investment advice.
- The site can reasonably expand into related tools later.
